Я в процессе fullstack-разработки. Работаю над проектом Recipe-Kitchen-Share (Next.js App Router + TypeScript + Tailwind + shadcn/ui + Prisma + Neon/Postgres + Auth).

Задача
Нужно реализовать главную страницу Recipe-Kitchen-Share: это публичная страница с двумя секциями:
1) Недавние по дате создания по createdAt desc
2) Популярные по количеству лайков desc
На главной странице показываются только PUBLIC рецепты. Для авторизованных пользователей доступны дополнительные функции (создание/редактирование/удаление/переключение private/public). Также нужно добавить в (если не авторизован пользователь) поиск и не показывать кнопки редактирования/CRUD для промптов.

Компоненты в UI
- Верхний блок (Header):
  - Лого/название Recipe-Kitchen-Share (ссылка на гл/стр)
  - Навигация: главная, профиль, мои рецепты
  - Справа: кнопка вход/регистрация (в зависимости от статуса), темная тема, меню (если авторизован)
- Нижний блок (Footer):
  - Копирайт строка: (с) Recipe-Kitchen-Share + год
  - Ссылки: контакты, о нас
- Главная страница:
  - Hero-секция: заголовок + описание + кнопка начать работу
    - Кнопка на авторизацию: редирект на /auth/signin (или если уже авторизован) и показывать кнопку начать работу, если авторизован
  - Две секции рецептов:
    - Недавние (показывать 10
    - Популярные (топ 10 по likesCount)
  - Карточка рецепта в списке:
    - Заголовок, описание (короткое), теги
    - Лайк (если авторизован)
    - Ссылка на детали (переход на страницу /recipes/[id])
    - Кнопка редактирования + кнопка удаления (если владелец или админ)
    - Кнопка переключения видимости/публикации на главной
- Адаптивность: корректно на мобильных.

Реализация в коде
- Создать 2 запроса в БД (для каждой секции) через Prisma:
  - recentRecipes: where { isPublic: true } orderBy { createdAt: "desc" } take N
  - popularRecipes: where { isPublic: true } orderBy по likesCount desc take N
- Для подсчета likesCount использовать Prisma _count (или отдельную таблицу связей).
- Если пользователь авторизован, нужно в каждом рецепте добавить поле likedByMe (чтобы знать, лайкнул ли пользователь промпт).
  - Оптимизация для запросов: взять все recipeIds и найти Like, затем создать Set(recipeId).

- Создание/редактирование/удаление рецепта и переключение private/public должны быть доступны только авторизованным пользователям (переход на страницу /my).
